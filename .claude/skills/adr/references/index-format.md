# INDEX.md regeneration spec

The `adr` skill regenerates `context/adrs/INDEX.md` from on-disk frontmatter at the end of every `new` and `supersede` invocation. The index is *always* rebuilt — never edited surgically.

## Input

`Glob('context/adrs/0[0-9][0-9][0-9]-*.md')`. For each file, parse:

- The NNNN from the filename (the canonical identifier)
- The H1 title (the line starting `# NNNN — `)
- The blockquote summary (the first `> ...` line after the H1) — optional, only used for richer rendering if needed
- Frontmatter fields: `status`, `date`, `affects`, `intent`

## Sort order

By NNNN ascending (0001, 0002, …, 0017, 0018, …). Superseded ADRs stay in their original numerical position — they are not moved or hidden.

## Output shape

```markdown
# ADR index

Generated: YYYY-MM-DD. Regenerated by the `adr` skill on every `new` / `supersede` invocation.

| # | Title | Status | Date | Affects | Intent |
|---|---|---|---|---|---|
| [NNNN](NNNN-slug.md) | <H1 title without the `NNNN — ` prefix> | <status> | <date> | <affects> | <intent> |
| ...                                                                                                |
```

## Required columns

- **`#`** — markdown link to the ADR file, displayed as the NNNN
- **Title** — the H1 stripped of the `NNNN — ` prefix
- **Status** — frontmatter `status:` verbatim, including `Superseded-by NNNN` form
- **Date** — frontmatter `date:`, ISO `YYYY-MM-DD`
- **Affects** — frontmatter `affects:`
- **Intent** — frontmatter `intent:` (`concrete-need` / `learning` / `unknown`)

## Things this spec does NOT add

- No "summary" column. The blockquote summary is *only* for the SessionStart hook injection (which does its own rendering).
- No "removal-commits" or "supersedes" or "review-by" columns. Those are visible by clicking through to the ADR.
- No grouping headers. The table is a flat sort by NNNN.
- No status badges, emoji, or HTML. Plain markdown table only.

## Atomicity invariant

Every `new` and `supersede` invocation must end by overwriting `INDEX.md` with a freshly-generated table from disk. Partial writes are forbidden — if regeneration fails, the index is stale, and the next invocation will fix it.

There is no path where an ADR file exists without a corresponding row in `INDEX.md`, because the row is always derived from the file. The only failure mode is "the file was written and the index regeneration crashed before writing" — in which case re-running `/adr list` (which reads from disk, not from the index) is the recovery.

## What the SessionStart hook reads

`.claude/hooks/inject_adr_index.py` reads `INDEX.md` directly and injects a *trimmed* version into the session — typically just the table rows, capped at 40 lines. The hook does not parse frontmatter; the index *is* its data source. Therefore the table format above is also the contract with the hook — changing column order or adding columns will require updating the hook to match.
